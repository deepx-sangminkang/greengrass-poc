            Shell commands to enter directly on the core device
            
You can paste the commands below directly into the core device terminal and run them. Replace the <...> parts with actual values (or click "Generate install script" with the inputs above to produce a filled-in script).

            
            
# 0. Configure AWS credentials (required before installation)
export AWS_ACCESS_KEY_ID=<access-key>
export AWS_SECRET_ACCESS_KEY=<secret-key>
export AWS_SESSION_TOKEN=<session-token>   # if using temporary credentials
# or: aws configure

# 1. Check whether Java is installed (install if missing)
if ! command -v java >/dev/null 2>&1; then
  echo "Java is required before installing AWS IoT Greengrass Core."
  exit 1
fi

# 2. Download & unzip Greengrass Nucleus
curl -s https://d2s8p88vqu9w66.cloudfront.net/releases/greengrass-nucleus-latest.zip > greengrass-nucleus-latest.zip
rm -rf GreengrassInstaller
unzip -q greengrass-nucleus-latest.zip -d GreengrassInstaller

# 3. Install Greengrass Core (auto-provisioning + system service registration)
sudo -E java -Droot="/greengrass/v2" -Dlog.store=FILE \
  -jar ./GreengrassInstaller/lib/Greengrass.jar \
  --aws-region <region> \
  --thing-name <thing_name> \
  --thing-group-name <thing_group_name> \
  --tes-role-name <token_exchange_role_name> \
  --component-default-user 'ggc_user:ggc_group' \
  --provision true \
  --setup-system-service true \
  --deploy-dev-tools true

# 4. Check service status
sudo systemctl status greengrass.service --no-pager

          

          

            Detailed guide
            
The core device requires Java (JDK 8 or higher) and sudo privileges. The script first checks whether Java is installed with command -v java, and aborts immediately if it is not installed (it does not auto-install Java).

            
If Java is missing, install it on the device first.

            
# Debian/Ubuntu
sudo apt-get update && sudo apt-get install -y default-jdk

# Amazon Linux/RHEL
sudo yum install -y java-11-amazon-corretto-headless

            
The Greengrass installation steps are as follows.

            
    
              
  1. Download the Greengrass Nucleus zip and unzip it.
    2. 
              
  2. Run sudo -E java -jar Greengrass.jar ... with the --provision true option to automatically handle IoT Thing creation, certificate/key, policy attachment, and Greengrass Core registration.
    3. 
              
  3. Register the greengrass.service systemd service with --setup-system-service true, and deploy development tools with --deploy-dev-tools true.
    4. 
              
  4. The installation path is /greengrass/v2.
    5. 
            

            
⚠ Configure AWS credentials in advance. The install script does not include static keys such as AWS_ACCESS_KEY_ID/AWS_SECRET_ACCESS_KEY. The --provision true provisioning uses the credentials from the core device-side environment variables passed through sudo -E, so you must configure the credentials on the device before running the installation.

            
# Run on the core device before installation
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
export AWS_SESSION_TOKEN=...   # if using temporary credentials
# or: aws configure

            
Once installation completes, the runtime components use temporary credentials issued by the Token Exchange Service (TES) instead of static keys.

          Advanced: Run installation via SSH
          
SSH installation sends the AWS credentials used by this server (backend) to the core device for use in --provision true provisioning. There is no need to configure credentials separately on the device. (Credentials are passed only via sudo bash standard input and are not exposed in the process list.)
