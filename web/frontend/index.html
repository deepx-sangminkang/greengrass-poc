                Manual installation guide
                
    
                    
  1. Enter the Thing name. The Thing group and token exchange role use the stack outputs.
    2. 
                    
  2. Click "Generate install script" to create the Greengrass Core install script.
    3. 
                    
  3. Copy the generated script and run it on the core device terminal.
    4. 
                    
  4. Once installation completes, click "List existing Cores" to verify registration.

                    Shell commands to enter directly on the core device
                    
Paste the commands below into the core device terminal. Replace the <...> parts with actual values (or use "Generate install script" above to produce a filled-in script).

                    
                    
# 0. Configure AWS credentials (required before installation)
export AWS_ACCESS_KEY_ID=<access-key>
export AWS_SECRET_ACCESS_KEY=<secret-key>
export AWS_SESSION_TOKEN=<session-token>   # if using temporary credentials
# or: aws configure

# 1. Check whether Java is installed (install if missing)
if ! command -v java >/dev/null 2>&1; then
  echo "Java is required before installing AWS IoT Greengrass Core."
  exit 1
fi

# 2. Download & unzip Greengrass Nucleus
curl -s https://d2s8p88vqu9w66.cloudfront.net/releases/greengrass-nucleus-latest.zip > greengrass-nucleus-latest.zip
rm -rf GreengrassInstaller
unzip -q greengrass-nucleus-latest.zip -d GreengrassInstaller

# 3. Install Greengrass Core (auto-provisioning + system service registration)
sudo -E java -Droot="/greengrass/v2" -Dlog.store=FILE \
  -jar ./GreengrassInstaller/lib/Greengrass.jar \
  --aws-region <region> \
  --thing-name <thing_name> \
  --thing-group-name <thing_group_name> \
  --tes-role-name <token_exchange_role_name> \
  --component-default-user 'ggc_user:ggc_group' \
  --provision true \
  --setup-system-service true \
  --deploy-dev-tools true

# 4. Check service status
sudo systemctl status greengrass.service --no-pager

                

                

                    Detailed guide
                    
The core device requires Java (JDK 8 or higher) and sudo privileges. The script checks whether Java is installed with command -v java and aborts if it is not (it does not auto-install Java).

                    
# Debian/Ubuntu
sudo apt-get update && sudo apt-get install -y default-jdk

# Amazon Linux/RHEL
sudo yum install -y java-11-amazon-corretto-headless

                    
⚠ Configure AWS credentials in advance. The install script does not include static keys. The --provision true provisioning uses the credentials from the core device-side environment variables passed through sudo -E, so configure credentials on the device before running the installation.

                Advanced: Run installation via SSH
                
SSH installation sends the AWS credentials used by this server to the core device for --provision true provisioning. Credentials are passed only via sudo bash standard input and are not exposed in the process list.
